## Novedades en v2.0

### Generación Inteligente de Sistemas de Diseño

La característica principal de v2.0 es el **Generador de Sistema de Diseño** - un motor de razonamiento impulsado por IA que analiza los requisitos de tu proyecto y genera un sistema de diseño completo y personalizado en segundos.

### Cómo Funciona la Generación del Sistema de Diseño

```
┌─────────────────────────────────────────────────────────────────┐
│  1. SOLICITUD DEL USUARIO                                       │
│     "Construye una landing page para mi spa de belleza"         │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  2. BÚSQUEDA MULTI-DOMINIO (5 búsquedas paralelas)               │
│     • Coincidencia de tipo de producto (100 categorías)        │
│     • Recomendaciones de estilo (67 estilos)                   │
│     • Selección de paleta de colores (96 paletas)               │
│     • Patrones de landing page (24 patrones)                    │
│     • Combinación de tipografía (57 combinaciones de fuentes)    │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  3. MOTOR DE RAZONAMIENTO                                       │
│     • Coincidir producto → reglas de categoría UI               │
│     • Aplicar prioridades de estilo (ranking BM25)              │
│     • Filtrar anti-patrones para la industria                   │
│     • Procesar reglas de decisión (condiciones JSON)            │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  4. SALIDA COMPLETA DEL SISTEMA DE DISEÑO                      │
│     Patrón + Estilo + Colores + Tipografía + Efectos            │
│     + Anti-patrones a evitar + Lista de verificación pre-entrega│
└─────────────────────────────────────────────────────────────────┘
```

### 100 Reglas de Razonamiento Específicas por Industria

El motor de razonamiento incluye reglas especializadas para:

| Categoría | Ejemplos |
|----------|----------|
| **Tecnología y SaaS** | SaaS, Micro SaaS, B2B Enterprise, Herramientas de Desarrollo, Plataforma de IA/Chatbot |
| **Finanzas** | Fintech, Banca, Cripto, Seguros, Dashboard de Trading |
| **Salud** | Clínica Médica, Farmacia, Odontología, Veterinaria, Salud Mental |
| **E-commerce** | General, Lujo, Marketplace, Caja de Suscripción |
| **Servicios** | Belleza/Spa, Restaurante, Hotel, Legal, Consultoría |
| **Creativo** | Portafolio, Agencia, Fotografía, Juegos, Streaming de Música |
| **Tecnología Emergente** | Web3/NFT, Computación Espacial, Computación Cuántica, Sistemas Autónomos |

Cada regla incluye:
- **Patrón Recomendado** - Estructura de la landing page
- **Prioridad de Estilo** - Mejores estilos de UI coincidentes
- **Estado de Ánimo de Color** - Paletas apropiadas para la industria
- **Estado de Ánimo de Tipografía** - Coincidencia de personalidad de fuente
- **Efectos Clave** - Animaciones e interacciones
- **Anti-Patrones** - Qué NO hacer (por ejemplo, "gradientes púrpura/rosa de IA" para banca)

## Instalación

### Usando Claude Marketplace (Claude Code)

Instala directamente en Claude Code con dos comandos:

```
/plugin marketplace add nextlevelbuilder/ui-ux-pro-max-skill
/plugin install ui-ux-pro-max@ui-ux-pro-max-skill
```

### Usando CLI (Recomendado)

```bash
# Instalar CLI globalmente
npm install -g uipro-cli

# Ir a tu proyecto
cd /ruta/a/tu/proyecto

# Instalar para tu asistente de IA
uipro init --ai claude      # Claude Code
uipro init --ai cursor      # Cursor
uipro init --ai windsurf    # Windsurf
uipro init --ai antigravity # Antigravity
uipro init --ai copilot     # GitHub Copilot
uipro init --ai kiro        # Kiro
uipro init --ai codex       # Codex CLI
uipro init --ai qoder       # Qoder
uipro init --ai roocode     # Roo Code
uipro init --ai gemini      # Gemini CLI
uipro init --ai trae        # Trae
uipro init --ai opencode    # OpenCode
uipro init --ai continue    # Continue
uipro init --ai codebuddy   # CodeBuddy
uipro init --ai droid       # Droid (Factory)
uipro init --ai all         # Todos los asistentes
```

### Otros Comandos CLI

```bash

uipro update                # Actualiza a la última versión
uipro init --offline        # Omite descarga de GitHub, usa recursos incluidos
```

## Uso

### Modo Habilidad (Activación Automática)

**Soportado:** Claude Code, Cursor, Windsurf, Antigravity, Codex CLI, Continue, Gemini CLI, OpenCode, Qoder, CodeBuddy, Droid (Factory)

La habilidad se activa automáticamente cuando solicitas trabajo de UI/UX. Simplemente chatea de forma natural:

```
Construye una landing page para mi producto SaaS
```

> **Trae**: Cambia primero al modo **SOLO**. La habilidad se activará para solicitudes de UI/UX.

### Modo Flujo de Trabajo (Comando Slash)

**Soportado:** Kiro, GitHub Copilot, Roo Code

Usa el comando slash para invocar la habilidad:

```
/ui-ux-pro-max Construye una landing page para mi producto SaaS
```

### Ejemplos de Prompts

```
Construye una landing page para mi producto SaaS

Crea un dashboard para analítica de salud

Diseña un sitio web de portafolio con modo oscuro

Haz una UI de aplicación móvil para e-commerce

Construye una aplicación bancaria fintech con tema oscuro
```

### Cómo Funciona

1. **Tú pides** - Solicita cualquier tarea de UI/UX (construir, diseñar, crear, implementar, revisar, corregir, mejorar)
2. **Sistema de Diseño Generado** - La IA genera automáticamente un sistema de diseño completo usando el motor de razonamiento
3. **Recomendaciones inteligentes** - Basado en tu tipo de producto y requisitos, encuentra los mejores estilos, colores y tipografías coincidentes
4. **Generación de código** - Implementa la UI con colores, fuentes, espaciados y mejores prácticas adecuados
5. **Verificaciones pre-entrega** - Valida contra anti-patrones comunes de UI/UX

### Stacks Soportados

La habilidad proporciona pautas específicas por stack para:

| Categoría | Stacks |
|----------|--------|
| **Web (HTML)** | HTML + Tailwind (predeterminado) |
| **Ecosistema React** | React, Next.js, shadcn/ui |
| **Ecosistema Vue** | Vue, Nuxt.js, Nuxt UI |
| **Otras Web** | Svelte, Astro |
| **iOS** | SwiftUI |
| **Android** | Jetpack Compose |
| **Multiplataforma** | React Native, Flutter |

Solo menciona tu stack preferido en el prompt, o deja que se use por defecto HTML + Tailwind.

## Comando de Sistema de Diseño (Avanzado)

Para acceso directo al generador de sistema de diseño:

> Nota: Si instalaste a través de Continue, reemplaza `.claude/skills/` con `.continue/skills/` en los comandos siguientes. Para Droid (Factory), usa `.factory/skills/`.

```bash
# Generar sistema de diseño con salida ASCII
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "beauty spa wellness" --design-system -p "Serenity Spa"

# Generar con salida Markdown
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "fintech banking" --design-system -f markdown

# Búsqueda específica por dominio
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "glassmorphism" --domain style
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "elegant serif" --domain typography
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "dashboard" --domain chart

# Pautas específicas por stack
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "form validation" --stack react
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "responsive layout" --stack html-tailwind
```

### Persistir Sistema de Diseño (Patrón Maestro + Anulaciones)

Guarda tu sistema de diseño en archivos para **recuperación jerárquica entre sesiones**:

```bash
# Generar y persistir en design-system/MASTER.md
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "SaaS dashboard" --design-system --persist -p "MyApp"

# También crear un archivo de anulación específico por página
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "SaaS dashboard" --design-system --persist -p "MyApp" --page "dashboard"
```

Esto crea una estructura de carpetas `design-system/`:

```
design-system/
├── MASTER.md           # Fuente de Verdad Global (colores, tipografía, espaciado, componentes)
└── pages/
    └── dashboard.md    # Anulaciones específicas por página (solo desviaciones del Maestro)
```

**Cómo funciona la recuperación jerárquica:**
1. Al construir una página específica (por ejemplo, "Checkout"), primero verifica `design-system/pages/checkout.md`
2. Si el archivo de página existe, sus reglas **anulan** el archivo Maestro
3. Si no, usa exclusivamente `design-system/MASTER.md`

**Prompt de recuperación consciente del contexto:**
```
Estoy construyendo la página [Nombre de Página]. Por favor lee design-system/MASTER.md.
También verifica si existe design-system/pages/[nombre-de-pagina].md.
Si el archivo de página existe, prioriza sus reglas.
Si no, usa exclusivamente las reglas del Maestro.
Ahora, genera el código...
```

## Arquitectura y Contribuciones

### Para Usuarios

La base de código ha sido reestructurada para usar un **sistema de generación basado en plantillas**. Todos los archivos específicos por plataforma (`.cursor/`, `.windsurf/`, `.kiro/`, `.factory/`, etc.) ahora se generan dinámicamente por el CLI.

**Siempre usa el CLI para instalar:**

```bash
npm install -g uipro-cli
uipro init --ai <plataforma>
```

Esto asegura que obtengas las últimas plantillas y la estructura de archivos correcta para tu asistente de IA.

### Para Contribuidores

Si deseas contribuir a este proyecto:

```bash
# 1. Clonar el repositorio
git clone https://github.com/nextlevelbuilder/ui-ux-pro-max-skill.git
cd ui-ux-pro-max-skill

# 2. Entender la estructura
src/ui-ux-pro-max/           # Fuente de verdad (datos, scripts, plantillas)
cli/                         # Instalador CLI (genera archivos desde plantillas)
.claude/                     # Desarrollo local/prueba para habilidad de Claude Code
.factory/                    # Desarrollo local/prueba para habilidad de Droid (Factory)

# 3. Hacer cambios en src/ui-ux-pro-max/
# - data/*.csv              → Archivos de base de datos
# - scripts/*.py            → Motor de búsqueda y sistema de diseño
# - templates/              → Plantillas específicas por plataforma

# 4. Sincronizar con CLI y probar localmente
cp -r src/ui-ux-pro-max/data/* cli/assets/data/
cp -r src/ui-ux-pro-max/scripts/* cli/assets/scripts/
cp -r src/ui-ux-pro-max/templates/* cli/assets/templates/

# 5. Construir y probar CLI
cd cli && bun run build
node dist/index.js init --ai claude --offline  # Probar en carpeta temporal

# 6. Crear PR (nunca hacer push directamente a main)
git checkout -b feat/your-feature
git commit -m "feat: descripción"
git push -u origin feat/your-feature
gh pr create
```
